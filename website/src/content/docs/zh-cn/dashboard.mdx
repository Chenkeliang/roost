Roost 自带一个**本地** Web 仪表盘,用于可视化管理你的配置。用以下命令启动:

```sh
roost serve
```

它会启动一个本地 JSON API 服务器(默认端口 `4317`)来提供 Web UI。你可以让它指向特定的
配置仓库或 Web 构建:

```sh
roost serve --port 4317 --repo /path/to/config-repo --web packages/web/dist
```

<Aside type="note" title="仅在本地">
仪表盘完全运行在你的机器上。和 Roost 的其余部分一样,它没有服务器、没有账号、也没有
遥测——什么都不会发往任何地方。
</Aside>

## 导航

仪表盘使用**左侧分组侧边栏**而非顶部标签页。侧边栏分为三个区域:

**顶部**
- **概览** — 主力与从机的状态卡、漂移摘要、快捷的 Capture / Load 操作,以及各模块的
  健康状态点。

**模块组** — 每个 SyncModule 对应一个专属页面:
- **Dotfiles** — 已管理的 dotfile 列表,支持筛选与扫描添加。
- **软件包** — 你的 Brewfile(Formulae / Casks / App Store),支持从本机导入。
- **项目** — 已管理的 git 仓库,以及按远程主机分组的按需扫描结果。
- **应用配置** — 已管理的 `defaults` 域名,支持扫描添加。
- **别名与环境变量** — 以搜索为主的面板,管理别名、环境变量、PATH 条目和函数。

**系统项**(分隔线以下):
- **漂移** — 主力与从机状态的并排 diff。
- **时间线** — 快照历史,带预览与回滚。
- **设置** — 仓库路径、age 密钥、已注册模块、Git 远程与同步、隐私摘要和文档链接。

侧边栏底部有一个 **Docs** 链接和 **EN / 中** 语言切换器,可在英文与简体中文之间切换界面。

## 内容优先,按需发现

页面会立即加载已管理的内容——它不会在页面加载时扫描磁盘或调用外部工具(如 `brew`)。
扫描始终是**显式触发**的:你需要点击一个按钮,例如 **[扫描 git 项目]** 或 **[扫描 dotfiles]**。
这意味着无论你的主目录有多大,仪表盘始终保持快速响应。

底层设计原则是:启动时运行轻量的 `index()`(统计已管理的内容数量),只有在你主动请求时
才运行 `discover()` 遍历文件系统。详见源码仓库中的 ADR-0006。

## 模块页面

### Dotfiles

以可筛选的列表展示已管理的文件,按类型分类(shell、git、编辑器、其他)。点击 **[扫描 dotfiles]**
可发现更多候选项;每一项都可以一键添加。

### 软件包

解析你的 `roost/Brewfile` 并按分区展示:Formulae、Casks、App Store、Taps。如果未安装
Homebrew,页面会诚实地显示空白状态。如果还没有 Brewfile,会出现 **[从本机导入]** 按钮,
点击后自动为你执行 capture。

### 项目

显示已管理仓库的数量。点击 **[扫描 git 项目]** 可遍历主目录查找 git 仓库。结果按**远程主机
分组**显示(例如 `github.com`、`gitlab.yourcompany.com`、`no-remote`)。每行显示相对于
主目录的路径和远程 URL,并提供两个操作:

- **[测试]** — 对远程运行 `git ls-remote` 以检查可达性。在 VPN 受控的网络环境中很有用;
  结果会内联显示。
- **[保存]** — 将该项目添加到 `selection.yaml`。

路径以相对于主目录的格式存储(`~/…`),以确保跨机器的可移植性。

### 应用配置

以可筛选的列表展示已管理的 `defaults` 域名。点击 **[扫描]** 可发现最近发生变化的域名;
每个域名都可以单独添加。已知存放凭证的域名会被自动跳过。

### 别名与环境变量

以搜索为主的面板:输入即可筛选别名、环境变量、PATH 条目和函数。可内联添加或编辑条目。
密钥环境变量的值绝不显示;只展示引用定位符。

## 设置——Git 远程与同步

**设置**页面新增了 **Git 远程与同步** 区域,显示:

- **远程** — 为配置仓库配置的 git 远程 URL。
- **分支** — 当前分支,以及相对于远程的超前/落后提交数。
- **Push / Pull** — 将本地提交推送到远程或从远程拉取更新的按钮,附带内联的成功/错误输出。

如果未配置远程,该区域显示占位符,Push/Pull 按钮处于禁用状态。

仪表盘正在积极开发中。可用的视图与面板遵循已锁定的设计,但当前构建中可能尚未全部接通。
</Aside>
