**skills** 模块备份你的 agent *技能*,并跨 IDE(Claude Code、Codex、Gemini CLI、OpenCode)分发。

## 工作方式

- 技能的权威副本放在源目录(默认 `~/.agents/skills/`);每个技能是一个文件夹(约定含 `SKILL.md`)。
- Roost 把受管技能复制进仓库 `skills/`,并以**软链**(默认)或**拷贝**分发到各启用的 IDE。
- 技能可按 IDE 分别启用;矩阵每格一个标记 —— 已链接为 coral 对勾、冲突为警告、未启用为空心圈。

## 导入技能

打开 **Skills → Discover**,使用导入面板:

- **从本地文件夹**:粘贴或浏览到机器上的路径(Roost 拷贝进来;原始文件夹不受影响)。
- **从 `.zip`**:拖到拖拽区,或点击选择文件。
- **从 git 地址**:粘贴 GitHub(或任意 git)URL。

Roost 会**先扫描**来源并列出找到的技能 —— 单个技能(含 `SKILL.md` 的文件夹)或整个技能包(向下递归查找几层)。勾选要导入的,再应用。

导入与捕获走**同一道安全闸**:Secret Scanner 与体积上限会拦截有风险的内容,且**只拷贝文件、绝不执行**(第三方技能可能含脚本)。`.git` 不会被拷入。

## 冲突

若某 IDE 的技能目录已存在你自己的(非 Roost)目录,Roost 不会覆盖。**备份后接管**会把你的目录移到 `~/.roost-backups/`,再把 Roost 的副本链接到原位。

## 由别的工具管理的 skill

如果某个 skill 挂载是软链,且目标在 Roost 源目录之外 —— 例如 [cc-switch](https://github.com/farion1231/cc-switch)
这类运行时管理器从 `~/.cc-switch/` 分发 skill —— Roost 会标为**外部管理**(中性徽章,不是冲突)并放任不管。
识别是通用的:任何非 Roost 的软链挂载都会被认出来,内置的小注册表只负责给个友好名字(可经 `roost/external-managers.yaml` 扩展)。

当一个 Roost *曾经*管理的挂载被别的工具接管时,该行给两个明确选项 —— **重新接管**(备份对方的挂载并重新链接 Roost 的副本)
或 **让给 &lt;工具&gt;**(Roost 停止向该目标分发;其内容仍在备份中)。Roost 绝不自动覆盖别的管理器的文件 —— 由你决定。
