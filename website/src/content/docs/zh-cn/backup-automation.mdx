title: 备份自动化

---

Roost 能自己让备份保持最新,并在需要你处理时提醒你 —— 同时确保仓库永不膨胀。

## 自动备份

在**设置**里设定 Roost 运行期间多久捕获一次变化:

- **关闭** —— 只在你点**捕获**时备份。
- **每天**(默认)/ **每周** —— Roost 在后台自动捕获。

**自动推送**是另一个默认关闭的开关:开启后,一次成功的自动捕获也会推送到远端。推送为单飞 —— API 与调度器共用一把锁,两次推送绝不相撞。

无实际变化的捕获**不产生提交**(也不增长仓库)—— 只刷新「上次备份」时间戳。

## 保鲜提醒

**总览**仅在有事可做时按优先级显示横幅:

| 横幅 | 触发时机 | 操作 |
|---|---|---|
| **有新版本** | 发布了更新的 Roost 版本 | 打开 release |
| **仓库较新** | 自你上次同步后,另一台机器推送过 | 拉取 |
| **有未推送改动** | 本地提交还没上远端 | 推送 |
| **备份已过期** | 超过 7 天没备份 | 捕获 |

「上次备份」是最近一次捕获**运行**的时间(哪怕没有变化);**时间线**显示内容真正变化的时刻。两者回答不同的问题 ——「多久没检查了?」与「内容多久没变了?」。

## 更新检查

启动时 Roost 可检查 GitHub 是否有更新版本(可选,设置里开关)。这是应用唯一的对外请求 —— 不上报任何关于你的信息,无遥测。也可在设置里手动检查。

## 大文件防护

把大文件备进 git,会让此后每次改动都重新存整份,撑大仓库。Roost 对此设防 —— 且**绝不触碰你的本地文件**:

- **新的 >10MB 文件先拦下。** 捕获会拦住并询问:**仍要备份**(加入允许名单)或**移出备份**(从仓库移出并排除 —— 本地文件原封不动)。
- **存量大文件会被提示。** 总览列出已在备份中的大文件,每个提供**保留备份**(不再提醒)或**移出备份(不影响本地文件)**。

每一步处置都只作用于仓库侧:「移出备份」经 chezmoi 把文件从源仓库 forget;你的本机副本绝不被修改或删除。

## 不再有重新加密的膨胀

age 加密是非确定性的 —— 天真地重新加密一个没变的文件,会在每次捕获产生全新 blob,让仓库无限增长。Roost 记录每个加密文件明文的哈希,**内容没变就跳过重新加密**,因此对一个没变的环境反复捕获不会给仓库增加任何东西。
