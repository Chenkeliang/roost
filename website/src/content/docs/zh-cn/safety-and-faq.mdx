## 安全模型

- **默认 dry-run。** `roost load` 只预览改动。在你传 `--apply` 之前,什么都不会写入。
- **覆盖前先备份。** 应用时,Roost 会在替换前备份任何已有文件(例如放到
  `~/.roost-backups/` 下)。生成的 `env.sh` 及其备份都保持 `0600` 权限。
- **无云。** 你的配置存放在你自己的私有 git 仓库里。没有服务器、没有账号、也没有遥测。
- **密钥受门控。** `capture` 会运行一个明文密钥扫描器;密钥用你的 age 密钥加密,绝不在
  UI 或日志中显形。

<Aside type="caution" title="移除项不会重写历史">
`unmanage` 把某项从工作树中取出,但不会清除 git 历史。若某个被移除的文件曾经存放过密钥,
请轮换它,并用 `git filter-repo` 或 BFG 之类的工具清除历史。
</Aside>

## 常见问题

### 我需要密码管理器吗?

**不需要。** 多数场景——dotfiles、软件包、应用配置、项目——完全不需要密钥。若你确实要
备份密钥值,你可以在机器之间手动拷贝 age 密钥;1Password 和 rbw 只是可选的便利项。

### 我的数据会上传到任何地方吗?

**不会。** Roost 没有服务器、也没有遥测。你的配置仓库及其 remote 完全在你的掌控之下。
仪表盘也运行在本地。

### 如果某个密钥在新 Mac 上无法解析会怎样?

Roost 是失败安全的:它会跳过那一个值、向你发出警告(只指明变量名、绝不指明值),
并写入一个占位符而非半截的赋值。`roost doctor` 会预先标出缺失的依赖。

### Roost 会编辑我的 `.zshrc` 吗?

只会为受管的 `env.sh` 追加一行单一、幂等的 source,并用标记包裹。它绝不重排或重写你
文件的其余部分,解除管理时也会把这段块再次移除。

### 应用提示"无法验证开发者",打不开怎么办?

Roost 桌面应用(.dmg)**未经签名**——Apple 未对其进行公证。macOS Gatekeeper 会在首次启动时拦截未签名的应用。两种打开方式:

- **右键打开:** 在 Finder 中,右键点击(或按住 Control 单击)Roost 应用,选择**打开**,在弹出的对话框中确认。只需操作一次。
- **移除隔离属性:** 若右键方式不奏效,运行:
  ```sh
  xattr -d com.apple.quarantine /Applications/Roost.app
  ```

CLI 与引擎从源码构建,不受 Gatekeeper 影响。

### 支持哪些平台?

v1 仅支持 macOS。
