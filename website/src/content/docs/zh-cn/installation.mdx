<Aside type="caution" title="仅支持 macOS(v1)">
Roost v1 **仅面向 macOS**,没有跨平台构建。
</Aside>

Roost v0.1.0 是首个公开版本。**桌面应用**以 macOS `.dmg` 形式提供,支持 Apple Silicon 和 Intel;**CLI / 引擎**目前仍从源码运行(尚无已发布的 CLI 二进制)。

## 桌面应用

适用于 **Apple Silicon** 和 **Intel** 的预构建 `.dmg` 已发布到
[Releases](https://github.com/Chenkeliang/roost/releases) —— 选择与你 Mac 匹配的版本。打开 `.dmg` 后把 **Roost** 拖进「应用程序」。

<Aside type="note" title="改为从源码构建">
如果你偏好自行构建桌面应用(需 Rust 工具链):

```sh
pnpm install
pnpm build:desktop
```

产物在 `packages/web/src-tauri/target/release/bundle/`(`Roost.app` + `Roost_<version>_aarch64.dmg`)。
</Aside>

<Aside type="caution" title="首次启动——未签名">
Roost 尚未做 Apple 签名/公证,首次打开会被 Gatekeeper 拦下。
右键 `Roost.app` →「打开」→「打开」,或运行
`xattr -dr com.apple.quarantine /Applications/Roost.app`。
</Aside>

## 前置依赖

CLI 编排一小组受信任的外部工具。请按你打算使用的模块安装对应工具:

- **[chezmoi](https://www.chezmoi.io/)**——管理你的 dotfiles 与 age 加密。
- **[Homebrew](https://brew.sh/)**——软件包模块据此采集你的 `Brewfile`(formulae、cask 与 `mas` 应用)。
- **[mise](https://mise.jdx.dev/)**——软件包模块据此采集语言/运行时版本。
- **[age](https://github.com/FiloSottile/age)**——仅当你备份**密钥**值时才需要。

以及一套用于从源码构建的 Node.js 工具链:

- **Node.js 24**(见 `.nvmrc`)
- **pnpm**(仓库通过 `packageManager` 锁定版本)

`roost doctor` 会报告这些工具在你机器上的存在情况。

## 从源码构建

<Steps>

1. 克隆引擎仓库:

   ```sh
   git clone https://github.com/Chenkeliang/roost.git
   cd roost
   ```

2. 安装依赖并构建所有包:

   ```sh
   pnpm install
   pnpm -r build
   ```

3. (可选)运行测试套件:

   ```sh
   pnpm test
   ```

</Steps>

## 运行 CLI

构建后的 CLI 位于 `packages/cli/dist/index.js`。在 `roost` 二进制发布之前,用 Node
调用它:

<Tabs>
  <TabItem label="从源码运行">
    ```sh
    node packages/cli/dist/index.js doctor
    node packages/cli/dist/index.js --help
    ```
  </TabItem>
  <TabItem label="将来(已发布)">
    ```sh
    roost doctor
    roost --help
    ```
  </TabItem>
</Tabs>

为简洁起见,本文档其余示例都用 `roost <命令>`;从源码运行时,请替换为
`node packages/cli/dist/index.js <命令>`。

## 常用环境变量

- `ROOST_REPO`——你的配置仓库路径(默认为 chezmoi 的源目录 `~/.local/share/chezmoi`)。
- `ROOST_LOCALE`——CLI 语言(默认 `en`)。
