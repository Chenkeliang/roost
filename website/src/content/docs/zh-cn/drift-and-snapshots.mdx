每次 `capture` 都在你的配置仓库记一条提交,所以仓库的 git 历史**就是**你的快照历史。
**时间线**把它变成可浏览、可预览、可恢复的变更日志。

## 可读的变更日志提交

捕获不再统一提交「roost: capture」。每条提交的标题说明改了什么,正文逐文件列出:

```
capture: dotfiles(2) aitools(1) packages(1)

dotfiles: ~/.zshrc, ~/.npmrc (encrypted)
aitools: ~/.claude/CLAUDE.md
packages: Brewfile
```

恢复操作提交为 `restore: <文件> @ <sha7>`。整段历史读起来就像一份「你究竟做了什么」的日志。

## 时间线 —— 什么变了、何时变的

**时间线**按时间倒序列出快照 —— 短 SHA、标题、相对时间。点击某行可展开正文(逐文件清单)。
它读取你配置仓库的提交日志;首次捕获前显示「暂无快照」空状态。

## 文件级历史与恢复

在时间线顶部输入路径(如 `~/.zshrc`)并点**查看历史**,即可切到该文件的历史。每个旧版本都提供
**恢复此版本到仓库**;最新一条标为「当前版本」(恢复成自己无意义)。

恢复**仅作用于仓库侧** —— 它改写**仓库**副本并提交;本机文件原封不动。成功提示会带一个跳到
[同步复核](/zh-cn/sync-review/)的入口,在那里该文件显示为「仓库较新」,你经正常闸门应用。
由于每次恢复本身都是一条新提交,所以无需确认弹窗 —— 历史就是后悔药。

命令行:

```sh
roost history ~/.zshrc          # 列出该文件的快照
roost restore ~/.zshrc <sha>    # 把仓库副本恢复到该快照
```

## 内联文件预览

在 [AI 工具](/zh-cn/ai-tools/)页和 [Dotfiles](/zh-cn/modules/)页,点文件名即可内联预览内容
(只读、文本最多 256 KB)。加密项、凭据文件、以及任何被[密钥扫描器](/zh-cn/secrets/)命中的文件**绝不**显示。

## 漂移 —— 哪里不同步

漂移是仓库与本机此刻的差异,归入[同步复核](/zh-cn/sync-review/):**逐项**视图按条目处置,
**原始 diff** 视图按模块给出 `+`/`-` 着色的统一 diff。命令行:

```sh
roost status   # 各模块漂移概况:已同步 / 漂移 / 冲突
roost diff     # 每个漂移模块一份统一 diff
roost list     # 已管理项及其漂移状态
```

<Aside type="tip">
同步复核比较的是*此刻仓库 vs 本机*;时间线是*仓库随时间的历史*。两者合起来回答「这台 Mac 同步了吗?」
与「仓库是怎么变成现在这样的?」。
</Aside>
