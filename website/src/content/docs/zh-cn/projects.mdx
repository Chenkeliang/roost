**项目**模块记录你本地的 git 仓库,让你在新机器上重新克隆完整的工作集。

## 工作原理

初次使用时,模块只知道 `selection.yaml` 中已有的内容(轻量索引)。磁盘扫描是**按需触发**
的:在仪表盘点击 **[扫描 git 项目]**,或运行:

```sh
roost discover projects
```

扫描会遍历你的主目录,查找 `.git` 目录。

## 按远程主机分组

扫描完成后,发现的仓库按**远程主机分组**展示。例如:

- `github.com (12)` — GitHub 上的公开或私有仓库
- `gitlab.yourcompany.com (3)` — 内部 GitLab 实例
- `no-remote (2)` — 没有配置远程的仓库

使用列表顶部的主机筛选标签,可以聚焦到某个分组。

## 测试与保存

每行发现的仓库提供两个操作:

- **[测试]** — 对远程 URL 运行 `git ls-remote` 以验证可达性。结果(可达/不可达及消息)
  内联显示。在 VPN 受控网络或 SSH 密钥受限的环境中,这有助于在提交列表前确认访问权限。
- **[保存]** — 将仓库添加到 `selection.yaml`。已保存的仓库在从机运行 `roost load` 时
  会被重新克隆。

保存前不必先测试。测试仅供参考。

## 相对于主目录的路径

项目路径以相对于主目录的格式存储——例如 `~/code/my-project` 而非
`/Users/alice/code/my-project`。即使不同机器的用户名不同,条目也能保持有效。

## 诚实的离线/无认证处理

如果远程不可达(无 VPN、SSH 密钥缺失等),测试操作会显示 `git ls-remote` 返回的准确
错误信息,没有任何隐藏。你仍然可以保存该仓库——路径和远程会被记录;访问权限只在从机
重新克隆时才需要。

<Aside type="note">
项目模块记录远程以供重新克隆。它不同步工作树文件、分支或本地提交——那是 git 本身在
仓库克隆后的职责。
</Aside>
