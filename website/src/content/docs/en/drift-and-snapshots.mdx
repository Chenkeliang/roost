Every `capture` records a commit in your config repo, so the repo's git history
*is* your snapshot history. The **Timeline** turns that into a readable changelog
you can browse, preview, and restore from.

## Readable changelog commits

Captures no longer commit a uniform "roost: capture". Each commit's subject names
what changed and the body lists every file:

```
capture: dotfiles(2) aitools(1) packages(1)

dotfiles: ~/.zshrc, ~/.npmrc (encrypted)
aitools: ~/.claude/CLAUDE.md
packages: Brewfile
```

A restore commits as `restore: <file> @ <sha7>`. The whole history reads like a
log of what you actually did.

## Timeline — what changed, and when

The **Timeline** lists snapshots newest-first — short SHA, subject, relative time.
Click a row to expand its body (the per-file list). It reads your config repo's
commit log; before your first capture it shows a "No snapshots yet" empty state.

## Per-file history & restore

Type a path at the top of the Timeline (e.g. `~/.zshrc`) and **Show history** to
switch to that file's history. Each past version offers **Restore this version to
the repo**; the newest one is marked *current* (restoring to itself is a no-op).

Restore is **repo-side only** — it rewrites the *repo* copy and commits it; your
machine file is untouched. A success notice links you to [Sync Review](/sync-review/),
where the file now shows as *repo-newer* and you apply it through the normal gates.
Because every restore is itself a new commit, there's no confirmation dialog —
history is the undo.

On the CLI:

```sh
roost history ~/.zshrc          # list this file's snapshots
roost restore ~/.zshrc <sha>    # restore the repo copy to that snapshot
```

## Inline file preview

On the [AI Tools](/ai-tools/) and [Dotfiles](/modules/) pages, click a file's
name to preview its contents inline (read-only, text up to 256 KB). Encrypted
entries, credential files, and anything the [Secret Scanner](/secrets/) flags are
**never** shown.

## Drift — what's out of sync

Drift is the difference between the repo and the machine right now. It lives in
[Sync Review](/sync-review/): the **Items** view resolves per entry, and the
**Raw diff** view shows a unified, `+`/`-`-colored diff per module. On the CLI:

```sh
roost status   # per-module drift summary: synced / drift / conflict
roost diff     # unified diff per drifted module
roost list     # managed items plus their drift state
```

<Aside type="tip">
Sync Review compares *repo vs. machine right now*; the Timeline is *the repo's
history over time*. Together they answer "is this Mac in sync?" and "how did the
repo get here?".
</Aside>
