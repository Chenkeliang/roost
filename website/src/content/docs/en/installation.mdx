<Aside type="caution" title="macOS only (v1)">
Roost v1 targets **macOS only**. There are no cross-platform builds.
</Aside>

Roost v0.1.0 is the first public release. The **desktop app** ships as a macOS
`.dmg` for both Apple Silicon and Intel; the **CLI / engine** is still run from
source (no CLI binary published yet).

## Desktop app

Prebuilt `.dmg` files for **Apple Silicon** and **Intel** are published to
[Releases](https://github.com/Chenkeliang/roost/releases) — pick the one that
matches your Mac. Open the `.dmg` and drag **Roost** into Applications.

<Aside type="note" title="Build from source instead">
If you prefer to build the desktop app yourself (needs the Rust toolchain):

```sh
pnpm install
pnpm build:desktop
```

The app lands under `packages/web/src-tauri/target/release/bundle/`
(`Roost.app` + `Roost_<version>_aarch64.dmg`).
</Aside>

<Aside type="caution" title="First launch — unsigned">
Roost isn't Apple-signed/notarized yet, so Gatekeeper blocks the first open.
Right-click `Roost.app` → **Open** → **Open**, or run
`xattr -dr com.apple.quarantine /Applications/Roost.app`.
</Aside>

## Prerequisites

The CLI orchestrates a small set of trusted external tools. Install the ones for
the modules you intend to use:

- **[chezmoi](https://www.chezmoi.io/)** — manages your dotfiles and age encryption.
- **[Homebrew](https://brew.sh/)** — the package module captures your `Brewfile` (formulae, casks, and `mas` apps).
- **[mise](https://mise.jdx.dev/)** — language/runtime versions captured by the package module.
- **[age](https://github.com/FiloSottile/age)** — only needed if you back up **secret** values.

Plus a Node.js toolchain to build from source:

- **Node.js 24** (see `.nvmrc`)
- **pnpm** (the repo pins a version via `packageManager`)

`roost doctor` reports which of these are present on your machine.

## Build from source

<Steps>

1. Clone the engine repo:

   ```sh
   git clone https://github.com/Chenkeliang/roost.git
   cd roost
   ```

2. Install dependencies and build all packages:

   ```sh
   pnpm install
   pnpm -r build
   ```

3. (Optional) run the test suite:

   ```sh
   pnpm test
   ```

</Steps>

## Running the CLI

The built CLI lives at `packages/cli/dist/index.js`. Until a `roost` binary is
published, invoke it with Node:

<Tabs>
  <TabItem label="Run from source">
    ```sh
    node packages/cli/dist/index.js doctor
    node packages/cli/dist/index.js --help
    ```
  </TabItem>
  <TabItem label="Later (published)">
    ```sh
    roost doctor
    roost --help
    ```
  </TabItem>
</Tabs>

Examples in the rest of these docs use `roost <command>` for brevity; substitute
`node packages/cli/dist/index.js <command>` while running from source.

## Useful environment variables

- `ROOST_REPO` — path to your config repo (defaults to chezmoi's source dir, `~/.local/share/chezmoi`).
- `ROOST_LOCALE` — CLI language (`en` by default).
