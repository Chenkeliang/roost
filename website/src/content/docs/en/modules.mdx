## The module model

Every capability in Roost is a **module** that implements a common contract.
The core is deliberately empty of domain logic — new abilities are added as
modules, not as special cases sprinkled through the core. Each module knows how
to:

- **discover** candidates on the machine,
- report **status** / **diff** (drift between the repo and the machine),
- **capture** selected items into the config repo,
- **apply** the repo's state to the machine (dry-run aware), and
- **unmanage** an item.

You can list the currently registered modules:

```sh
roost plugins
```

## Built-in modules

### Dotfiles

Backs up existing config files as-is and restores them as files, orchestrated
through chezmoi. It deliberately skips sensitive paths (such as `~/.ssh/`,
`~/.aws/`, `~/.config/gh/`, `.npmrc`, `.git-credentials`, `.netrc`) and the
Roost-managed `~/.config/roost/` directory so it never double-manages what the
Aliases & Env module owns.

### Packages

Captures your software set into a `roost/Brewfile` via Homebrew — including
formulae, casks, and `mas` (Mac App Store) apps — plus language/runtime versions
via mise.

<Aside type="note">
`mas` only installs apps you've already purchased; it can't acquire new paid apps for you.
</Aside>

### App Config

Captures macOS application preferences using `defaults export` and restores them
with `defaults import` — **not** by symlinking plist files. Domains known to hold
credentials (OpenAI, Anthropic, GitHub, AWS, and similar) are skipped by default
so tokens aren't swept in. The `roost app learn` command helps you discover which
preference domains change when you toggle a setting in a GUI app.

### Projects

Discovers your local git repositories and records them so you can re-clone your
working set on a new machine. It can flag repos that have no remote or
uncommitted changes.

### Aliases & Env

Manages portable aliases, environment variables, PATH entries, and shell
functions as structured data, regenerating `~/.config/roost/env.sh` on each
machine. This is the one module that **owns** a slice of shell config rather than
copying a file. It gets its own page:
