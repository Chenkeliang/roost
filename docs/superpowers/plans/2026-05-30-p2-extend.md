# Roost P2 — Extend Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development / executing-plans. Steps use `- [ ]`.
> **粒度说明**:P2 任务级规范,**开工前先用 writing-plans 展开为 bite-sized TDD**(同 P1)。

**Goal:** 在 MVP 之上加 app 配置通用发现 + Learn Mode、projects + mise、导入器、密钥审计/轮换、插件加载——把"任意 app 也能纳管"和"生态可扩展"补齐。

**Architecture:** 续 P0/P1;新增能力全部走 `SyncModule` 或 §7 扩展点,不改 core 分支(I4)。

**前置**:P1 完成。**约束**:不偏离锁定基线;偏离须开 ADR。

---

### Task 1: appconfig 模块 — 通用发现(不靠固定注册表)
- **Files:** `packages/core/src/modules/appconfig.ts`、`packages/core/src/discovery/defaults.ts`(+tests)
- **做法:** `discover` 用 `defaults domains` + `~/Library/Containers/*/Data/.../Preferences` + `~/Library/Group Containers/*` + `~/Library/Application Support/*` + unix 风格;按 bundle id 关联;噪音/体积过滤;给建议(track/skip)。**不导出全量**,只列候选。
- **Tests:** mock `defaults domains` 输出 → 候选分类正确;过滤掉账号/机器相关域。
- **验收:** 在真机列出 ~30 个值得纳管的域(对照 inventory §4 夹具)。
- **Refs:** design §4/§10;architecture §10(macOS 硬约束)。

### Task 2: Learn Mode(`roost app learn`)
- **Files:** `packages/cli/src/commands/learn.ts`、`packages/core/src/discovery/learn.ts`(+tests)
- **做法:** 录制:快照全部 `defaults domains` → 提示用户去 GUI 改设置 → re-snapshot diff 出**变更 domain/key** → `defaults export` 文本入库 → 用户确认命名。
- **Tests:** 给两次快照 diff,断言识别出变更域;export 文本写入 selection/appconfig。
- **验收:** 能抓到一个 **mackup 不认识**的本地 app 的配置变更并纳管。
- **Refs:** design §11.2(Learn Mode)。

### Task 3: cfprefsd 安全 apply(§10 硬坑)
- **Files:** `packages/core/src/modules/appconfig-apply.ts`(+tests)
- **做法:** apply 经 `defaults import`/`write`(非拷文件);区分 **bootstrap(app 未运行可直拷)/ runtime(先 `osascript` 退 app 再 import)**;绝不 symlink plist。
- **Tests:** mock 场景断言走对路径;runtime 分支先退 app。
- **验收:** 在另一台还原 app 偏好且重启 app 后生效(真机验证)。
- **Refs:** design §10(坑一二三);architecture §10。

### Task 4: projects 模块 + mise(per-project env)
- **Files:** `packages/core/src/modules/projects.ts`(+tests)
- **做法:** `capture` 记 repo→路径 + envTool 到 `roost/projects.yaml`;`apply` clone/pull,写 `.mise.toml`(共享)且不覆盖 `.mise.local.toml`(本地机密);**无 remote 仓库检测并告警**;脏仓库提示先处理。
- **Tests:** mock git;无 remote/脏仓库进入告警分支;mise 文件分工正确。
- **验收:** 清单驱动在从机重建项目目录;per-project env 经 mise 生效(对照 inventory §6 夹具)。
- **Refs:** design §3(projects)/ADR-0001(mise)。

### Task 5: 导入器(降低上手门槛)
- **Files:** `packages/core/src/import/*.ts`(`Importer` 接口 + mackup / 既有 dotfiles 仓库 / 裸 git)(+tests)
- **做法:** `detect` 识别既有方案 → 产出 `selection.yaml` 草案供用户确认。
- **Tests:** 给样例 mackup `.cfg` / 既有 dotfiles 树 → 正确映射候选。
- **验收:** 从一个既有 dotfiles 仓库一键迁入并产出合理 selection。
- **Refs:** architecture §7(扩展点:导入器)。

### Task 6: 密钥审计 + 轮换
- **Files:** `packages/cli/src/commands/audit.ts`、`packages/core/src/secrets/rotate.ts`(+tests)
- **做法:** `audit` 报告仓库内密钥项与"是否都已加密";`key rotate` 一键 re-encrypt 全部密钥到新 age recipient。
- **Tests:** 审计发现遗漏明文;轮换后旧 key 不能解、新 key 能解。
- **验收:** 轮换不丢数据、全部密文可被新 key 解。
- **Refs:** design §14 P2;architecture §9。

### Task 7: 插件加载(`roost-module-*`)+ 示例插件
- **Files:** `packages/core/src/plugins/loader.ts`(+tests);示例仓库 `examples/roost-module-fonts/`
- **做法:** Plugin Loader 按 npm 包名约定发现、校验 manifest(name/version/deps/schemaVersion/兼容 core API 版本)、注册;**插件只走 core 公共 API,不得绕 exec/Secret Pipeline**。
- **Tests:** 加载示例插件成功注册;版本/能力不符被拒。
- **验收:** 第三方示例模块(fonts)装上即出现在 `list`/`doctor`,无需改 core。
- **Refs:** architecture §7(扩展规则)。

### Task 8: P2 验收
- **验收(P2 "Done"):**
  - 纳管一个 **mackup 不认识**的本地 app(经 Learn Mode)并在另一台还原成功。
  - projects 清单驱动重建 + mise per-project env 生效;无 remote/脏仓库被正确告警。
  - 一个第三方示例模块经插件加载注册,core 未改分支。
  - 密钥可审计、可轮换;CI(ubuntu+macos)全绿。
