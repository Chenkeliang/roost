# Roost P1 — MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development / executing-plans. Steps use `- [ ]`.
> **粒度说明**:本计划是 P1 的**任务级规范**(目标/文件/做法/验收/引用)。**每个任务在开工前先用 writing-plans 展开为 P0 同款的 bite-sized TDD 步骤**——这是刻意的"按需展开",避免对未动工的代码预写全量代码而产生 churn(符合"不过度设计")。

**Goal:** 让全新用户在主力机自选并 `capture` dotfiles/secrets/软件,从机一行 bootstrap + `load` 安全还原;敏感项 age 加密、提交前扫描、覆盖前可回退。

**Architecture:** chezmoi 为 dotfiles/加密引擎,Roost 编排其上(I1);所有外部命令经 exec 适配器(I3);能力以 `SyncModule` 插入(I4)。

**Tech Stack:** 续 P0 + chezmoi/age/sops、execa、`@clack/prompts`(向导 TUI)、密码管理器 CLI(op / rbw)。

**前置**:P0 完成(`p0-foundation` 标签)。**约束**:不偏离 architecture(LOCKED)/ ADR-0001;偏离须开 ADR。

---

### Task 1: chezmoi 适配器 + `roost init`
- **Files:** `packages/core/src/adapters/chezmoi.ts`(+test);`packages/cli/src/commands/init.ts`(+test)
- **做法:** 适配器封装 `chezmoi init/add/apply/diff/verify/managed`(经 exec)。`init` 连接或创建用户配置仓库(= chezmoi 源目录),写 `.chezmoi.toml.tmpl`(age recipient、profile 变量),建 `roost/` 子目录骨架。
- **Tests:** 适配器全程 mock exec;`init` 在临时 HOME 沙箱创建出预期目录结构。
- **验收:** 空机器 `roost init` 后得到合法 chezmoi 源 + `roost/selection.yaml` 空壳。
- **Refs:** design §5.2/§7;architecture §4–§6。

### Task 2: 密钥后端 + age key 托管(ADR-0001 两层模型)
- **Files:** `packages/core/src/secrets/backend.ts`(`SecretBackend` 接口 + `op`/`rbw` 实现)(+test);`packages/core/src/secrets/agekey.ts`(+test)
- **做法:** bootstrap 时从 1Password(`op read`)或 Bitwarden(`rbw get`)取 age 私钥 → `~/.config/sops/age/keys.txt`(chmod 600);生成并提示离线**恢复码**。后端只读注入,不落盘多余副本。
- **Tests:** mock `op`/`rbw` 输出;断言写入权限 600、缺失时报清晰错误。
- **验收:** 无 age key 的从机经后端拿到 key 后可解密往返。
- **Refs:** design §9;architecture §9。

### Task 3: Secret Pipeline + Scanner(capture 硬门,I6 / M7)
- **Files:** `packages/core/src/secrets/pipeline.ts`、`packages/core/src/secrets/scanner.ts`(+tests)
- **做法:** 分类(track/encrypt)→ 经 chezmoi `--encrypt`(age)加密 → 注入。Scanner 用 gitleaks 式规则扫描"将入库内容",**检出明文密钥即阻断 capture** 并提示改为 encrypt。
- **Tests:** 含已知 token 样本被拦截;纯文本配置放行;加密往返后明文不出现在仓库。
- **验收:** 故意把含 token 的文件标 track → capture 被拦;改 encrypt 后通过且仓库内为密文。
- **Refs:** design §9;architecture §9。

### Task 4: Selection Store(I5)
- **Files:** `packages/core/src/selection.ts`(+test)
- **做法:** 读写校验 `roost/selection.yaml`(带 `schemaVersion`);增删纳管项;校验未知模块/字段报错。
- **Tests:** 往返序列化稳定;坏 schema 报错;schemaVersion 缺失时迁移到当前。
- **验收:** `select` 与 `list` 共用此 store,改动可持久化。
- **Refs:** architecture §5–§6;ROADMAP 数据契约。

### Task 5: dotfiles 模块
- **Files:** `packages/core/src/modules/dotfiles.ts`(+tests)
- **做法:** `discover` 扫 `~` + `~/.config`(经 Discovery 噪音/体积守卫)→ 候选 + 建议分类;`capture` 经 chezmoi add(敏感 `--encrypt`);`apply` 经 chezmoi apply(dryRun 支持);`status` 经 chezmoi verify;`diff`/`unmanage`/`doctor`(检查 chezmoi 存在)。
- **Tests:** 临时 HOME 沙箱 + mock exec;capture→apply 幂等;discover 过滤缓存/大目录。
- **验收:** 选 `.zshrc` 等 → capture → 另一沙箱 apply 一致;`~/.config/env.sh` 走加密。
- **Refs:** design §1/§7;inventory/INVENTORY.md §1(夹具)。

### Task 6: packages 模块(Brewfile + mas)
- **Files:** `packages/core/src/modules/packages.ts`(+tests);仓库内 `run_onchange_before_install-packages.sh.tmpl`
- **做法:** `capture` = `brew bundle dump` 写 `roost/Brewfile`;`apply` 委托 chezmoi `run_onchange_`(内容变才 `brew bundle`);mas 项标注"需手动/已购";`doctor` 检 brew/mas。
- **Tests:** mock brew/mas;Brewfile 往返;mas 未登录给出可操作提示。
- **验收:** 主力 dump 的 Brewfile 在从机 `brew bundle` 还原;mas 限制如实提示。
- **Refs:** design §3/§10(坑四);inventory/Brewfile.draft(夹具)。

### Task 7: Snapshot/State 服务
- **Files:** `packages/core/src/state.ts`(+test)
- **做法:** 每次 capture/load 写 `state/<host>.json`(已装包/文件 hash/纳管项);每次 sync 一个 git commit(喂 P3 时间线/漂移)。
- **Tests:** 状态写出可读回;hash 稳定。
- **验收:** 两台机各有 state 文件,可比对(为 P3 漂移铺路)。
- **Refs:** design §8;architecture §6。

### Task 8: 安全可逆 apply(I7)
- **Files:** `packages/core/src/apply.ts`(+tests)
- **做法:** Planner 产 `ApplyPlan`;apply **默认 dryRun**;真实 apply 覆盖前对从机现有文件**先备份**(记 `backup` 路径)、检测从机本地改动并告警;支持逐项选择。
- **Tests:** dryRun 零写入;覆盖产生备份且可回退;本地有改动时进入告警分支。
- **验收:** load 覆盖前必有备份;`--dry-run` 只出计划不动盘。
- **Refs:** design §8;architecture §1(I7)。

### Task 9: 选择向导 + 预设
- **Files:** `packages/cli/src/wizard.ts`(+test);`packages/core/src/presets/*.ts`(数据)
- **做法:** `@clack/prompts` 勾选式:类别 → 候选项;首屏可选预设("Developer essentials" 等,纯数据)。文案经 `t()`(en/zh)。
- **Tests:** 给定 discover 结果,向导产出正确 selection;预设可加载合并。
- **验收:** 新用户不懂细节也能靠预设三步选完。
- **Refs:** design §7;frontend §3(选择语义)。

### Task 10: CLI 命令接线
- **Files:** `packages/cli/src/commands/{select,capture,load,list,status,diff}.ts`(+tests)
- **做法:** 把上述能力接到命令;`list` 展示已纳管 + 状态 + 可增删;`load` 默认 dry-run 预览。
- **Tests:** 每命令一条 happy-path(沙箱 + mock exec)。
- **验收:** `init→select→capture→(从机)load→list` 全链路在沙箱跑通。
- **Refs:** design §17 命令速览。

### Task 11: `bootstrap.sh`
- **Files:** `scripts/bootstrap.sh`
- **做法:** 纯 bash(先于 Node):装 Homebrew → `brew install chezmoi age node mise git` → 取 age key(Task 2)→ clone 配置仓库 → `npx roost load`。
- **Tests:** shellcheck;在干净容器/VM dry-run。
- **验收:** 一行命令从零到可用(真机/VM 验证)。
- **Refs:** design §7;ROADMAP lead-time(macOS runner)。

### Task 12: P1 端到端 + 验收
- **Files:** `packages/core/test/e2e-roundtrip.test.ts`
- **做法:** 临时 HOME 沙箱模拟"主力→从机"完整往返(dotfiles+secrets+Brewfile);在 macOS CI 跑。
- **P1 验收(MVP "Done"):**
  - **全新用户**(非作者)主力选若干项 → push;从机一行 bootstrap 从零到可用,**敏感项加密往返正确**;`list` 可见并增删。
  - 全程**明文密钥零泄露**(scanner 生效);**load 覆盖前有备份可回退**;`--dry-run` 默认预览。
  - CI(ubuntu+macos)全绿。
